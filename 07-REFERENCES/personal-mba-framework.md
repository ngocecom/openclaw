## 🎯 KNOWLEDGE CAPTURE PRINCIPLES

### The 5 Whys
When capturing information, ask "why" 5 times to get to root cause/insight.

### Feynman Technique
Explain concepts in simple language. If you can't explain it simply, you don't understand it well enough.

### Spaced Repetition
Review important concepts at increasing intervals (1 day, 1 week, 1 month, 3 months).

### Interleaving
Mix different types of learning/practice rather than focusing on one thing.

## 📝 DOCUMENTATION STANDARDS

### Every File Should Have:
1. **Title** - Clear, descriptive
2. **Date** - When created/updated
3. **Purpose** - Why this exists
4. **Context** - Background information
5. **Action Items** - What needs to be done
6. **References** - Links to related files

### File Naming Convention:
```
YYYY-MM-DD-topic-description.md
Example: 2026-03-12-cliproxy-setup-guide.md
```

### Tags/Categories:
- `#value-creation`
- `#marketing`
- `#sales`
- `#value-delivery`
- `#finance`
- `#human-systems`

## 🔄 WEEKLY REVIEW PROCESS

### Every Sunday (20:00 VN Time):
1. **Review** all files created this week
2. **Categorize** into 5 functions
3. **Archive** outdated information
4. **Update** MEMORY.md with key insights
5. **Plan** next week's priorities

### Questions to Ask:
- What did we learn?
- What worked well?
- What didn't work?
- What should we stop/start/continue?

## 📊 KNOWLEDGE METRICS

Track these monthly:
- Files created
- Files updated
- Time to find information
- Decision quality (review old decisions)
- Learning velocity (new concepts mastered)

## 🎓 LEARNING CYCLE

```
1. CAPTURE → Save information (files, notes)
2. ORGANIZE → Categorize by 5 functions
3. DISTILL → Extract key insights
4. APPLY → Use in real work
5. REVIEW → Reflect and update
```
